RF Engineer Internship  |  Parrot Drones (Feb – Aug 2025)

Automated four RF test benches and boosted measurement productivity by 70 % for Parrot’s next‑gen professional drone platforms.

Company & Mission

Parrot is Europe’s leading drone manufacturer, supplying lightweight, 4K‑enabled ANAFI systems to defence, public‑safety and inspection markets. My six‑month mission inside the Hardware RF team was clear: teach four heterogeneous benches — 5G, LoRa, Mars and GNSS — to run themselves.

Key Achievements

Automation Framework

Built Python + Tkinter / PyQt GUIs that replaced CLI scripts, added real‑time plots & server archiving, and cut manual test time > 70 %.

5G Bench Revamp

Added TRP mode, spherical scans (ϕ/θ), dynamic EPRE control and CSV + PDF report generation on a CMW500 / chamber rig.

Ground‑Up LoRa Bench

Designed hardware layout, USB‑controlled attenuator logic & PyQt interface for 2 D sensitivity sweeps at 868 / 915 MHz.

Mars Dual‑Mode Tests

Created conducted & radiated setups (anechoic and desktop box) inc. auto RX‑card calibration and 4 Mbps‑based stop criteria.

GNSS Co‑existence Bench

Profiled C/No degradation while toggling video, Wi‑Fi & Mars modules; delivered mitigation guidance to firmware team.

Extra Contributions

Repaired LT360 turntable, calibrated chambers via VNA/SA, and re‑designed a power‑optimised Mars reader PCB in OrCAD.

Tools & Stack

  • Instrumentation: R&S CMW500, Spectrum Analyzer, Keysight EXG, VNA
  • Control: SCPI over USB/TCP, ADB, PyVISA, bash
  • Software: Python 3, Tkinter, PyQt 5, Pandas, Matplotlib, OrCAD, Git
  • Environments: Full‑size + mini anechoic chambers, LT360 & Maturo turntables
